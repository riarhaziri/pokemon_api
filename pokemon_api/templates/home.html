Available endpoints and what they do:

/insertcsv

POST

Insert the pokemon.csv data located in the app.py
directory into the database

/deleteall

DELETE

Delete all the data located in the specified database

/view

GET

Expose(view) data from the database with or
without sorting, filtering and ordering that
should be given in the request body as json

Manipulating what you get from this endpoint is done through request's json body. If you
want to sort by any of the columns you can do that by using the key 'sort_by_column' and
the value of the column you want to sort by ex. 'type1' like this:

{
"sort_by_column" : "type1"
}



If you want the order to be ascending or descending you use the key 'order' and
the value 'asc' for ascending(defalut) or 'dsc' for descending:

{
"sort_by_column" : "type1",
"order" : "dsc"
}



to get the pokemons ordered by type1 in descending order.
If you want to filter the results from the database you use the key 'filter_by_type' and
the value of the type you want(ex. Fire, Water, Wind etc). This will show all the pokemos
that have the 'filter_by_type' matching either in type1 or in type2. The json looks like:

{
"filter_by_type" : "Fire"
}



Finally the way that the api was setup makes it possible for the user to use all three keys
at the same time and the api will return the data that is expected. One example of that would be

{
"filter_by_type" : "Water",
"sort_by_column" : "attack",
"order" : "dsc"
}


So using this json body the response would return pokemons that are Water type, sorted by the attack column
and ordered in descending order
